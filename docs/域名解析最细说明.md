# 域名解析最细说明

目标域名：xynong.ccwu.cc

## 1. 先确认你的域名在谁那里注册
你需要进入你的域名服务商后台，比如：
- 阿里云
- 腾讯云
- Cloudflare
- Namesilo
- 其他域名注册商

## 2. 进入域名解析设置页
在你的域名服务商后台找到：
- DNS / 域名解析 / DNS 管理

然后进入这个域名的解析列表。

## 3. 选择要添加的解析记录
针对你当前这个域名 xynong.ccwu.cc，建议直接使用 A 记录，因为它是根域名（apex domain），不是 www 子域名。

### 方案 A：A 记录（适合你当前域名）
这是最稳妥的做法：
- 类型：A
- 主机名/名称：@
- 值：185.199.108.153

再分别添加三个相同类型的记录：
- 185.199.109.153
- 185.199.110.153
- 185.199.111.153

### 方案 B：CNAME 记录（适合 www 子域名）
如果你想使用 www.xynong.ccwu.cc，而不是 xynong.ccwu.cc，那么用这个：
- 类型：CNAME
- 主机名/名称：www
- 值/目标：xynong.github.io

> 结论：你的当前域名 xynong.ccwu.cc 请选择 A 记录，不要用 CNAME。

## 4. 你需要填写的内容
按你的服务商界面，通常只要填这几个字段：
- 类型：CNAME 或 A
- 主机名 / 名称：www 或 @
- 记录值 / 目标：xynong.github.io 或 IP 地址
- TTL：默认即可，通常 600 / 300 / 3600 都可以

## 5. 保存后等待生效
解析生效通常需要：
- 5 分钟到 1 小时
- 有些服务商可能要更久

## 6. 然后回到 GitHub Pages 里填写
在 GitHub 仓库的 Pages 设置里：
- Custom domain 填：xynong.ccwu.cc
- 点击 Save

## 7. 最后访问测试
等一段时间后访问：
- https://xynong.ccwu.cc

## 8. 最短版操作
如果你想最省事，直接照着做：
1. 去域名服务商 → DNS 管理
2. 新建一条 CNAME
3. 主机名填 www
4. 目标填 xynong.github.io
5. 保存
6. 回 GitHub Pages 填 xynong.ccwu.cc
7. 等待生效
